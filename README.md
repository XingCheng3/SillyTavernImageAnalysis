# SillyTavern Image Analysis

> A local-first workspace for **SillyTavern character cards and lorebooks** — edit, translate, generate with AI, patch selectively, and export back to PNG.

一个面向 **SillyTavern 角色卡 / 世界书（Lorebook）** 的本地工作台。
它不是通用素材管理平台，也不是 SaaS；它更适合拿来做 **角色卡编辑、世界书整理、AI 辅助创作、翻译对比、PNG 导出闭环**。

## 这个项目是干什么的？

如果你经常在 SillyTavern 里处理这些事情：

- 想修改一张现成的 PNG 角色卡
- 想把角色卡里的设定、开场白、世界书做批量翻译
- 想先让 AI 生成世界书草稿，再自己审核后应用
- 想从一个提示词开始，直接做出整张角色卡
- 想保留与 SillyTavern 的结构兼容，而不是导出成一堆散乱文本

那这个项目就是为这类工作流准备的。

---

## 核心价值

### 1. 角色卡编辑不是“拆文件”，而是保留 SillyTavern 兼容结构
- 支持导入 **PNG 角色卡**
- 支持导入 **JSON 角色卡**
- 统一编辑 **基础信息 / 高级设置 / 世界书**
- 导出时兼容 `chara`、`ccv3`、`character_book`、`lore` 等结构
- 尽量贴近 SillyTavern 的角色卡与世界书数据逻辑

### 2. AI 不只是“直接生成一大坨文本”
项目当前已经内置更稳的 AI 工作流：
- **批量翻译**：基础字段 / 高级字段 / 世界书条目
- **翻译对比后再应用**：避免 AI 直接覆盖原文
- **世界书 AI 生成**：先生成结构化草稿，再应用到编辑区
- **世界书局部改写**：按条目 / 段落 / 字段做 patch，先预览 diff 再应用
- **从 0 创建角色卡**：支持从提示词直接生成角色卡草稿与世界书骨架/正文

### 3. 更适合“反复打磨”的本地工作流
- 快照 / 撤销 / 恢复初始
- 全局替换
- 替换或增加封面
- 导出回 PNG，继续在 SillyTavern 中使用

## 兼容与关注重点

当前项目重点对齐并兼容这些结构方向：

- `chara`
- `ccv3`
- `character_book`
- `lore`
- `greetings / first_mes / alternate_greetings`

目标不是“做一个自定义格式编辑器”，而是尽量保证：

> **在本工具里编辑过的角色卡，仍能顺畅回到 SillyTavern 工作流中。**

## 典型使用流程

### 场景 A：修改已有角色卡
1. 导入 PNG 或 JSON 角色卡
2. 编辑基础信息 / 高级设置 / 世界书
3. 如有需要，执行批量翻译或全局替换
4. 保存快照，确认结果
5. 导出回 PNG

### 场景 B：先让 AI 生成世界书，再人工审核
1. 导入角色卡
2. 打开“世界书 AI 生成”
3. 输入设定前提、条目数量、开场分支等信息
4. 生成结构化草稿
5. 预览并选择应用到当前编辑区
6. 导出角色卡

### 场景 C：从 0 创建角色卡
1. 点击“从 0 创建角色卡”
2. 输入核心设定、风格、语言、目标条目数等信息
3. 让 AI 生成角色卡草稿
4. 如部分条目失败，可按条目重试补全
5. 应用草稿到编辑区
6. 调整细节后导出 PNG

## 为什么不是普通翻译器？

这个项目和“把字段丢给 AI 然后直接回填”的工具不太一样。

它更强调：

- **结构化输入输出**，而不是只处理自由文本
- **先校验、再应用**，避免错误结果直接落盘
- **可预览、可回滚**，适合反复打磨设定
- **导出闭环**，不是编辑完就丢失 SillyTavern 兼容性

如果你要的是“快速改卡 + AI 辅助 + 最终还能稳定导回 ST”，它比普通文本编辑器更合适。

## AI 接口说明

本项目当前是 **纯前端本地项目**。
AI 请求由浏览器直接访问 **OpenAI 兼容接口**，需要你在页面设置中提供：

- API URL
- API Key
- Model

这意味着它非常适合本地自用或小范围使用；但如果未来要对外提供服务，建议增加：

- 服务端代理
- 限流 / 配额控制
- 请求审计
- 安全隔离
- 敏感凭据保护

## 项目结构入口

如果你准备继续开发，这几个入口最重要：

- `src/views/index.vue`：主工作台编排入口
- `src/utils/characterCardParser.js`：角色卡解析 / 导出核心

- `src/composables/useWorldBookAIGeneration.js`：世界书 AI 生成流程
- `src/composables/useWorldBookAIPatch.js`：世界书局部改写流程
- `src/composables/useCharacterAICreation.js`：从 0 AI 创建角色卡流程
